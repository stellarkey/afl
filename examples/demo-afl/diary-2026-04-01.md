# 2026-04-01 Work Log

First deployment to staging. **Severe failure!**

Container kept crash-looping within 30 seconds of startup.

Investigation:
1. Checked pod logs — database connection timeout
2. Checked K8s service — DB ClusterIP service was reachable
3. Root cause: connection pool overflow — Drizzle default pool_size=10, staging PG max_connections=20, 3 pods × 10 = 30 > 20

Fix: set `DB_POOL_SIZE=5` via env var, added connection retry logic.

Also discovered health check endpoint was missing — K8s liveness probe kept failing, causing continuous restarts. Added `/health` endpoint returning DB connection status.

See [[debug-deploy-failure]] for full debug trace.
See [[know-pg-tuning]] for connection pool formula.
See [[config-staging]] for staging DB constraints.

Lessons learned: @attr(severity: "critical", resolved: true)
- Connection pool size must account for replica count
- Always configure health check endpoints first
